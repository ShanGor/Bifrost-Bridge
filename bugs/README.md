## Bug Classification
- **Critical**: System crash or data loss
- **High**: Major functionality broken
- **Medium**: Feature not working as expected
- **Low**: Minor issues or improvements
- **Enhancement**: New feature requests or requirement bugs

## Reporting Process
1. Create a new bug file with format: `YYYY-MM-DD-short-description.md`
2. Use the provided template
3. Update the master index below
4. Assign severity and status
5. Track resolution in the bug file
